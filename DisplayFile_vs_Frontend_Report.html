7. Cross-Cutting Differences Summary

Aspect AS/400 Display File (DSPF) React Frontend
RenderingServer-rendered 5250 terminal stream, 24×80 fixed character gridClient-side SPA, responsive Material UI grid with CSS flexbox/grid
NavigationFunction keys (F3, F5, F6, F12) + typed selection codesReact Router v6 + clickable buttons / navigation cards
Input ModelTyped codes per field, Enter to submit all fields at onceIndividual form controls, click/type with live validation
ValidationServer-side only (RPG program logic after Enter)Dual: client-side Zod schema validation + server-side Spring validator
Error DisplaySingle error message line at fixed screen row positionPer-field inline helperText errors + floating Snackbar toasts
List Actions"Op" column: type 2/4/5 next to rows, Enter to batch-processPer-row icon buttons (Edit/View/Delete) with immediate async execution
Computed FieldsCalculated server-side, refreshed on EnterLive-updating on keystroke (e.g., ALG-001 gross margin)
StylingGreen-on-black monochrome, fixed-width fontDark glassmorphic theme, gradient icons, animations, color-coded status chips
State ManagementStateful server session (RPG program call stack)Stateless REST API + client-side React state (useState / react-hook-form)
PrintingOVRPRTF → spool file → output queueHTTP POST to print endpoint → backend generates spool
List/SubfileSFL/SFLCTL record formats with SFLSIZ/SFLPAG pagingArray.map() into <Table> rows, unlimited scroll
New Features—Urgency indicators, live margin color, skeleton loading states, responsive layout, inline mode-switch button, KPI summary cards
